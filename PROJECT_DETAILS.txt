# Indian Bovine Breed Recognition System (Production Edge Release 1.0)

## Project Overview
The **Indian Bovine Breed Recognition System** has evolved into a production-grade, offline-capable Edge AI product. It is designed for deployment in rural India to identify indigenous cattle and buffalo breeds without internet connectivity.

## Core Architecture
- **Edge Inference Engine**: TensorFlow Lite (MobileNetV2, Quantized).
- **Latency**: Sub-200ms inference on standard CPUs.
- **Offline Capability**: Fully functional in "Airplane Mode".
- **Knowledge Layer**: Embedded local JSON database for instant husbandry guidance.

## Key Features & Status
1. **Offline Inference (✅ IMPLEMENTED)**
   - Replaced PyTorch runtime with `tflite_runtime`.
   - Model converted to `.tflite` format.
   - All assets self-contained in `/models`.

2. **Lightweight CNN (✅ IMPLEMENTED)**
   - Architecture: **MobileNetV2** (Optimized for Mobile/Edge).
   - Replaced ResNet-50 for 10x smaller footprint and faster inference.
   - Retained accuracy via Transfer Learning.

3. **Knowledge Decision Support (✅ IMPLEMENTED)**
   - **Local Database**: `knowledge_base/breed_data.json` containing 41 breeds.
   - **Features**:
     - Region Mapping
     - Ideal Climate Analysis
     - Feed Recommendations
     - Disease Awareness (Health Notes)
   - **Disclaimer**: Included for veterinary compliance.

4. **Explainable AI (✅ RETAINED)**
   - **Grad-CAM**: Visualizes features for transparency.
   - **Hybrid Mode**: Uses TFLite for speed, Keras model for deep analysis when requested.

5. **Frontend Application (✅ DEPLOYED)**
   - **Framework**: React + Vite.
   - **Features**: Camera capture, Confidence bars, Top-3 Predictions.
   - **Status**: Visualizes all knowledge fields including Region.

## Technology Stack
- **AI/ML**: TensorFlow, TensorFlow Lite, Keras.
- **Backend**: Python (Flask).
- **Frontend**: React.js.
- **Data**: Local JSON & Compressed Image Assets.

## Dataset
- **Status**: Used strictly for training/validation. Not accessed at runtime.
- **Size**: ~4,000 images across 41 classes.
- **Imbalance Handling**: Data augmentation (Rotation, Zoom, Shear) applied during training.

## Deployment Instructions
1. **Training**: `python model_training/train_mobilenet_fixed.py` (Generates `/models`).
2. **Backend**: `python backend_api/app.py` (Starts Inference Server).
3. **Frontend**: `npm run dev` (Starts User Interface).

## Reliability
- **Confidence Threshold**: < 45% confidence is flagged or rejected as "Not Bovine" to avoid false certainty.
- **Error Handling**: Graceful degradation if camera or model fails.

This system is now positioned as a **Real World Edge Product**, ready for pilot testing in rural veterinary centers.
